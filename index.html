Right-click blocked — the context menu ("Save image as…") is completely disabled on the whole page.

Drag & drop blocked — users can't drag images out of the browser into their desktop or another app.

Keyboard shortcuts blocked — Ctrl+S (save page), Ctrl+P (print), Ctrl+U (view source), F12 and devtools shortcuts are all disabled.

CSS pointer-events: none — images have no mouse interaction at all, so browser-native image menus won't appear.

Transparent overlay — every image gets an invisible ::after pseudo-element on top, acting as a glass shield that intercepts any clicks/touches before they reach the actual image.

How it works:
Every image's base64 data is XOR-scrambled (each character shifted) and then reversed, so the string looks like noise. A tiny JS decoder runs silently at page load to unscramble it back — completely invisible to the visitor, the site works normally.

What this protects against:
